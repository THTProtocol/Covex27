# Contributing to Covex

Covex is an honest, pragmatic covenant explorer and visibility platform for Kaspa. The project values technical accuracy over marketing claims.

## Philosophy

1. **Radical honesty**: Label aspirational features as "design targets" or "not yet implemented." Never present simulations as real.
2. **Pragmatic progress**: Ship what works today. Document what will work tomorrow.
3. **No middlemen**: Covens are non-custodial. Keys never leave the user's wallet.

## What We Need

- **Real ZK circuits**: More circuits beyond MerkleMembership (range proofs, verifiable compute stubs)
- **On-chain integration**: Help improving silverc to support richer covenant ops
- **Frontend polish**: The React + Tailwind frontend is functional but always improving
- **Documentation**: Clear guides, integration examples, operational runbooks
- **Testing**: Production validation, load testing, abuse scenarios

## Getting Started

1. Read `docs/BUILDING_ON_COVEX.md` for integration patterns
2. Review the architecture notes in the README and skill documentation
3. Check the open issues on GitHub
4. Run `./deploy/validate-production.sh` to verify your setup

## Pull Requests

- Keep PRs focused. One feature or fix per PR.
- Update documentation alongside code changes.
- Be honest about limitations in PR descriptions.
- No simulated/fake features — if it's aspirational, label it.
