Covex cold recovery

Claim covenant funds with no Covex frontend and no Covex backend in the loop. Your private key stays in your browser, is never transmitted, and is cleared after signing.

Open the tool

A single self-contained HTML page with the Kaspa signing library inlined. It assembles, signs and broadcasts (or exports) the redeem transaction that releases your funds. Once loaded you can save it and run it from file:// on an air-gapped machine.

Open cold recovery tool

What it actually claims (honest)

The chain (or a revealed public secret plus a key's signature check) enforces only some spend paths end to end. The tool is explicit about which:

  • Script-enforced primitives and revealed-secret branches are offline-claimable with your own key.
  • The non-refundable WIN path of the oracle kinds still needs the Covex oracle's half-signature, so it is not offline-claimable here.
  • The refundable kinds' refund branch is offline-claimable by the refund key after the on-chain lock elapses.

No Covex involvement is required for the offline-claimable paths.

Verify before you trust it

A tool that signs with your private key should be verified, not trusted on faith. After loading, save the page and compare its SHA-256 against the value published in the build output and the guide. The signing logic is byte-identical to the audited in-app redeemer.
